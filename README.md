# QA Manual Testing: Requirements Analysis & Test Design (Urban Routes)

## 📌 Project Overview
This repository contains the QA documentation and test design artifacts for **Urban Routes**, a web application that simulates a ride-hailing and logistics service. The main objective of this project was to analyze functional requirements, design rigorous test cases using industry-standard techniques, and document software defects found during execution.

---

## 🛠️ Skills & Concepts Applied
* **Requirements Analysis:** Breaking down software specifications to identify ambiguities, gaps, and testing scopes.
* **Test Design Techniques:** Implementing **Equivalence Partitioning (EP)** and **Boundary Value Analysis (BVA)** to optimize test coverage.
* **QA Documentation:** Creating comprehensive Test Cases, Checklists, and Bug Reports.
* **Defect Lifecycle Management:** Reporting and categorizing bugs based on severity and priority.

---

## 📂 Test Design & Strategy

### 1. Requirements Analysis
Before writing any test cases, a deep dive into the application specifications was performed. This ensured that all user flows (such as ordering a taxi, adding a credit card, or booking a delivery) met the expected business logic.

### 2. Test Design Techniques
To avoid exhaustive testing and ensure efficiency, the following logic-based exercises were applied to input fields (e.g., phone number validation, credit card details, and distance logic):
* **Equivalence Partitioning (EP):** Dividing input data into valid and invalid partitions.
* **Boundary Value Analysis (BVA):** Testing the edges of valid and invalid limits (e.g., minimum and maximum character lengths, numeric boundaries).
